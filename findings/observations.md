## Observations

During the lab exercise, network traffic was monitored and analyzed to understand TCP connection behavior and traffic patterns within a controlled environment.

### Network Discovery

- Active hosts on the subnet were successfully identified using network discovery techniques.
- Each discovered host responded as expected, allowing baseline visibility of the lab environment.
- IP addressing and device identification were verified before traffic analysis began.

### Traffic Analysis

- Packet captures revealed normal TCP communication between devices on the network.
- TCP SYN packets were observed during connection establishment.
- The TCP three-way handshake process was successfully identified and analyzed.
- Source and destination IP addresses were clearly visible within packet captures.
- Traffic patterns could be tracked and correlated to specific devices on the network.

### Wireshark Findings

- Wireshark provided detailed visibility into packet-level communications.
- TCP traffic was the most frequently observed protocol during testing.
- Packet timestamps helped identify communication sequences and connection attempts.
- Network conversations and endpoints were successfully mapped.

### Security Perspective

- Packet analysis demonstrated how network monitoring can be used to identify unusual communication patterns.
- Understanding TCP behavior is essential for detecting abnormal network activity.
- Baseline traffic analysis improves the ability to recognize suspicious behavior during future investigations.
- Proper network visibility is critical for incident response and security monitoring.

### Lessons Learned

- Gained practical experience with network discovery and packet analysis.
- Improved understanding of TCP/IP networking fundamentals.
- Developed familiarity with Wireshark filtering and traffic inspection.
- Strengthened technical documentation and analytical skills through hands-on lab work.
